Sobre o Projeto ACTx

Visão Geral

O ACTx é um projeto desenvolvido para demonstrar habilidades técnicas em desenvolvimento de software, com foco em uma arquitetura monolítica com Web Service RESTFull para simular sistemas legados em evolução. O projeto utiliza uma combinação de tecnologias modernas e consolidadas para criar uma solução robusta e escalável.

Arquitetura do Sistema

A arquitetura do sistema é dividida em duas principais camadas: o back-end monolítico, que utiliza a plataforma Jakarta EE 10, e o front-end, que possui uma interface em Angular v19 e JSF (Jakarta Server Faces). A camada de serviços é implementada com EJB (Enterprise JavaBeans), enquanto a persistência de dados é gerenciada por Hibernate com JPA e PostgreSQL.

Tecnologias e Ferramentas

  • Back-End: Jakarta EE 10, JSF, EJB, JAX-RS, Hibernate, JPA, PostgreSQL, WildFly, Spring Framework (Spring Boot, Spring MVC, Spring Security).
  • Front-End: Angular v19, Thymeleaf, JSF.

Objetivos e Funcionalidades

O objetivo principal do ACTx é oferecer uma base sólida para a evolução de sistemas legados, permitindo a abstração e transição para microserviços conforme necessário. O projeto inclui funcionalidades chave como:

  • Comunicação eficiente e segura entre o back-end e o front-end.
  • Gestão de estados para manter a consistência e integridade dos dados.
  • Otimização da performance do sistema para lidar com um grande número de requisições.
  • Implementação robusta de segurança com Spring Security.

Equipe e Contribuições

O desenvolvimento do ACTx contou com uma equipe dedicada, responsável pela implementação de diversas partes do sistema. As contribuições incluem:

  • Desenvolvimento da comunicação entre o back-end monolítico e o front-end Angular utilizando API RESTful.
  • Implementação da camada de serviços com EJB e segurança com Spring Security.
  • Persistência de dados com Hibernate e JPA em PostgreSQL.
  • Desenvolvimento de testes unitários e de integração para garantir a qualidade do software.
